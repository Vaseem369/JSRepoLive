Day 3 — Operators & Core JS Concepts

Topic 1 Arithmetic (+ - * / % **)

The basics: addition, subtraction, multiplication, division, remainder, and exponent; increment and decrement; then a small GST example on a fixed price.

Example code 

// The basics
console.log(10 + 3);                 // 13   addition
console.log(10 - 3);                 // 7    subtraction
console.log(10 * 3);                 // 30   multiplication
console.log(10 / 3);                 // 3.3333333...  division
console.log(10 % 3);                 // 1    modulo (remainder)
console.log(10 ** 3);                // 1000 exponent (10^3)

// Increment / decrement
let count = 5;
count++;                             // count = count + 1
console.log(count);                  // 6
count--;                             // count = count - 1
console.log(count);                  // 5

// Real-world: GST calculator
const price = 1000;
const gst = price * 18 / 100;        // 180
const total = price + gst;           // 1180
console.log(`Total with GST: ₹${total}`);    // "Total with GST: ₹1180"

                        Topic 2
                    

                    
                        String + Coercion (+ concatenates · others convert)
                    

                

                


                    

                        The
                        +
                        operator prefers string concatenation if
                        either
                        side is a string. Operators like
                        -,
                        *,
                        /,
                        %
                        try to convert operands to numbers.
                    


                    

                        Example code
                    


                    
// Pure math
console.log(2 + 3);                  // 5    (number + number)

// Pure concatenation
console.log("Hi, " + "Priya");       // "Hi, Priya"   (string + string)

// Mixed — coercion surprise
console.log("Age: " + 25);           // "Age: 25"     (number → string)
console.log(25 + "");                // "25"          (number → string)
console.log("5" + 1);                // "51"          (number → string!)
console.log("5" - 1);                // 4             (string → number, because - has no string version)
console.log("5" * 2);                // 10            (string → number)


                    

                        Reference output (console order)
                    


                    


                    

                        Rule of thumb:
                        +
                        prefers concatenation if a string is involved;
                        -
                        *
                        /
                        %
                        coerce to numbers.
                    


                    

                        Form input bug:
                        reading values from a form gives you a
                        string,
                        even when it looks like a number.
                        "5" + 1 === "51",
                        not
                        6.
                        Convert with
                        Number(input)
                        (or similar) before math.
                    


                    

                        Follow along — three results?
                    


                    
console.log(1 + 2 + "3");
console.log("1" + 2 + 3);
console.log("5" - 2);

                        Topic 3
                    

                    
                        Assignment (= += -= *= /= %= **=)
                    

                

                


                    

                        Compound assignments are shorthand for updating a
                        variable from its current value, including
                        **=
                        for exponents. The
                        +=
                        operator also appends to strings.
                    


                    

                        Example code
                    


                    
let x = 10;            // simple assignment

x += 5;                // same as: x = x + 5  → x is 15
x -= 3;                // same as: x = x - 3  → x is 12
x *= 2;                // same as: x = x * 2  → x is 24
x /= 4;                // same as: x = x / 4  → x is 6
x %= 4;                // same as: x = x % 4  → x is 2
x **= 3;               // same as: x = x ** 3 → x is 8

// String compound — also works
let greeting = "Hello";
greeting += ", Priya";
console.log(greeting);               // "Hello, Priya"


                    

                        Reference output (console order)
                    


                    


                


            


            

            


                

                    
                        Topic 4
                    

                    
                        Comparison (< > <= >=)
                    

                

                


                    

                        Numeric comparisons return booleans. String
                        comparisons use lexicographic (alphabetical) order.
                        Mixed operands are coerced to numbers when possible.
                    


                    

                        Example code
                    


                    
// Numeric comparisons return booleans
console.log(5 > 3);                  // true
console.log(5 < 3);                  // false
console.log(5 >= 5);                 // true
console.log(5 <= 4);                 // false

// String comparisons — alphabetical (lexicographic)
console.log("apple" < "banana");     // true   ('a' < 'b')
console.log("Z" < "a");              // true   (uppercase < lowercase in ASCII!)

// Mixed — JS converts both to numbers
console.log("10" > 5);               // true   ("10" → 10)
console.log("abc" > 5);              // false  ("abc" → NaN, all comparisons with NaN are false)


                    

                        Reference output (console order)
                    


                    


                


            


            

            


                

                    
                        Topic 5
                    

                    
                        == vs === (always use ===)
                    

                

                


                    

                        The rule:
                        prefer
                        ===
                        and
                        !==.
                        Loose
                        ==
                        applies coercion rules that are easy to get wrong;
                        modern JavaScript style uses strict equality everywhere.
                    


                    

                        Example code
                    


                    
// Same type — both work the same
console.log(5 === 5);                // true
console.log("a" === "a");            // true

// Different types — STRICT
console.log(5 === "5");              // false   (number vs string)
console.log(0 === false);            // false   (number vs boolean)
console.log(null === undefined);     // false   (different types)

// Different types — LOOSE (coercion happens)
console.log(5 == "5");               // true    (string converted to number)
console.log(0 == false);             // true    (both → 0)
console.log("" == false);            // true    (both → 0)
console.log(null == undefined);      // true    (special rule)

// Inequality has both versions too
console.log(5 !== "5");              // true   (strict not-equal)
console.log(5 != "5");               // false  (loose not-equal — they're "equal")


                    

                        Reference output (console order)
                    


                    


                


            


            

            


                

                    
                        Topic 6
                    

                    
                        Logical (&& · || · !)
                    

                

                


                    

                        AND requires both sides to be truthy for a true result;
                        OR needs at least one truthy; NOT flips a boolean.
                        &&
                        and
                        ||
                        can short-circuit and sometimes return a
                        value,
                        not only
                        true
                        /
                        false.
                    


                    

                        Example code
                    


                    
// AND — both must be true
console.log(true && true);           // true
console.log(true && false);          // false

// OR — at least one must be true
console.log(true || false);          // true
console.log(false || false);         // false

// NOT — flips the boolean
console.log(!true);                  // false
console.log(!false);                 // true

// Real-world: combining conditions
const age = 25;
const hasEmail = true;

const canRegister = age >= 18 && hasEmail;
console.log(canRegister);            // true

const isMinor = age < 18;
const isInvited = false;
const cannotEnter = isMinor || !isInvited;
console.log(cannotEnter);            // true   (because !isInvited is true)

// Short-circuit — return the actual value, not just true/false
const userName = null;
const displayName = userName || "Guest";       // null is falsy → "Guest"
console.log(displayName);                       // "Guest"

// && returns the first falsy OR the last value
console.log("Priya" && "Aarav");                // "Aarav"  (both truthy → last one)
console.log(0 && "anything");                    // 0        (first falsy → 0)


                    

                        Reference output (console order)
                    


                    


                


            


            

            


                

                    
                        Topic 7
                    

                    
                        Ternary (condition ? a : b)
                    

                

                


                    

                        A ternary is a one-line
                        if / else
                        expression. It works well in template literals for
                        small branches. Avoid deep nesting — refactor to
                        if
                        when readability drops (Day 4 covers
                        if / else
                        in depth).
                    


                    

                        Example code
                    


                    
const age = 17;

// Long way (Day 4 will cover if/else)
let category;
if (age >= 18) {
  category = "adult";
} else {
  category = "minor";
}
console.log(category);               // "minor"

// Ternary — one line, same result
const cat = age >= 18 ? "adult" : "minor";
console.log(cat);                    // "minor"

// Practical use: in template literals
const fee = age >= 65 ? 0 : 100;
console.log(`Entry fee: ₹${fee}`);   // "Entry fee: ₹100"

// Don't nest ternaries deep — readability tanks
// const status = a ? b ? c : d : e ? f : g;   ← BAD, refactor with if/else
