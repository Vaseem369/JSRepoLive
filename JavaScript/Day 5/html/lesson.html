Day 5 — Loops & Iteration

Topic 1 The Classic for Loop — init ; condition ; update

Visual reference 

Syntax:   for (init; condition; update) { ...body }

Order:   init runs once → check condition → run body → update → check condition → repeat.

Part         Example      When
init         let i = 0    Once at start
condition    i < 5      Before each iteration
update       i++          After each iteration

Follow along — predict each output 

for (let i = 0; i < 5; i++) console.log(i);
→
your answer

for (let i = 5; i > 0; i--) console.log(i);
→
your answer

for (let i = 0; i <= 10; i += 2) console.log(i);
→
your answer

                        Topic 2
                    

                    
                        while & do...while
                    

                

                


                    

                        while (cond) { ... }
                        checks the condition
                        first
                        — the body might never run.
                        do { ... } while (cond);
                        runs the body
                        first
                        , then checks — always at least one iteration.
                    


                    

                        Critical:
                        something inside the loop must eventually make the
                        condition false — otherwise you get an infinite loop.
                    


                    

                        Visual reference
                    


                    
while (cond) { ... }     — condition first, then body
do { ... } while (cond); — body first, then condition


                    

                        Follow along
                    


                    
let i = 0;
while (i < 3) { console.log(i); i++; }
→
your answer

let n = 10;
do { console.log(n); n++; } while (n < 5);
→
your answer


                    

                        Reference answers
                    


                    


                


            


            

            


                

                    
                        Topic 3
                    

                    
                        break & continue
                    

                

                


                    

                        break
                        exits the loop entirely.
                        continue
                        skips the rest of the current iteration and jumps to
                        the next one.
                    


                    

                        Visual reference
                    


                    
break    — exit the whole loop immediately
continue — skip to the next iteration


                    

                        Follow along
                    


                    
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  console.log(i);
}
→
your answer

for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) continue;
  console.log(i);
}
→
your answer


                    

                        Reference answers
                    


                    


                


            


            

            


                

                    
                        Topic 4
                    

                    
                        for...of — Arrays & Strings
                    

                

                


                    

                        for (const value of iterable) { ... }
                        — modern iteration over
                        values.
                        Works on arrays, strings, Maps, Sets, and other iterables.
                    


                    

                        Use
                        const
                        in the head: a fresh binding each iteration.
                    


                    

                        Visual reference
                    


                    
for (const value of iterable) { ...body }


                    

                        Follow along
                    


                    
const fruits = ["apple", "banana", "mango"];
for (const f of fruits) console.log(f);
→
your answer

for (const ch of "Priya") console.log(ch);
→
your answer


                    

                        Reference answers
                    


                    


                


            


            

            


                

                    
                        Topic 5
                    

                    
                        for...in — Object Keys
                    

                

                


                    

                        for (const key in object)
                        gives you each
                        property name
                        (key). Read values with
                        obj[key].
                    


                    

                        Do not use for...in on arrays
                        — keys are strings, inherited properties can appear, and
                        for...of
                        is the right tool for array values.
                    


                    

                        Memory aid:
                        in
                        for
                        keys,
                        of
                        for
                        values.
                    


                    

                        Visual reference
                    


                    
for (const key in object) { ...body }


                    

                        Follow along
                    


                    
const s = { name: "Aarav", age: 22 };
for (const k in s) console.log(k);
→
your answer

for (const k in s) console.log(k, s[k]);
→
your answer


                    

                        Reference answers
                    


                    


                


            


            

            


                

                    
                        Topic 6
                    

                    
                        Avoiding Infinite Loops
                    

                

                


                    

                        Three classic ways to freeze the tab: forgetting to
                        update the counter, updating the wrong way, or a
                        condition that never becomes false
                        (e.g.
                        while (true)
                        without a break).
                    


                    

                        If the tab freezes: close it. Everyone hits this once
                        — it is a rite of passage.
                    


                    

                        Spot the bug
                    


                    
// Why is this infinite?
let i = 0;
while (i < 5) {
  console.log(i);
  // ???

}
