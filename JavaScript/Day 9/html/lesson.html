Day 9 — ES6+ Essentials

Template literals, array and object destructuring, rest vs spread, object merge patterns, || vs ??, and computed property names — with reference output per topic.

Topic 1 Template literals

Backticks `...` — three strengths: interpolation with ${expression}, multiline strings without \n, and any embedded expression (calls, arithmetic, ternaries).

Example 

const n = 3;
const block = `Line one
count = ${n * 2}`;

                        Array destructuring
                    

                

                


                    

                        Quick patterns
                    


                    
const [a, b, c] = arr;           // basic
const [, , last] = arr;         // skip slots
const [x, y, z = 9] = short;    // defaults
[x, y] = [y, x];                // swap
const [head, ...tail] = arr;    // rest (array)

                        Topic 3
                    

                    
                        Object destructuring (deeper)
                    

                

                


                    

                        Forms
                    


                    
Basic    const { name, age } = user
Rename   const { name: who } = user
Default  const { phone = "N/A" } = user
Nested   const { address: { city } } = user   // city only, not address
Rest     const { name, ...rest } = user
In param ({ name, age }) => ...


                    

                        Nested gotcha:
                        const { address: { city } } = user
                        binds
                        city
                        only — it does
                        not
                        create an
                        address
                        variable.

                        Topic 4
                    

                    
                        Rest parameters
                    

                

                


                    

                        ...args
                        in a
                        parameter list
                        collects remaining arguments into a real array. Rest must
                        be
                        last.
                        Contrast with spread: spread
                        expands;
                        rest
                        collects.

                        Topic 5
                    

                    
                        Spread — arrays & objects
                    

                

                


                    

                        Array spread concatenates or forwards arguments; object
                        spread copies enumerable own properties and is ideal for
                        merge with override
                        (later keys win).

                        Topic 6
                    

                    
                        Defaults: || vs ??
                    

                

                


                    

                        Rules
                    


                    
||  → falls back when left is ANY falsy value
??  → falls back only when left is null or undefined

                        Topic 7
                    

                    
                        Computed property names
                    

                

                


                    

                        Inside an object literal,
                        [expression]
                        becomes the key name — no separate assignment step.
                    


                    

                        Example
                    


                    
const fieldName = "email";
const value = "priya@x.in";

const user2 = {
  name: "Priya",
  [fieldName]: value,
  [`is_${fieldName}_verified`]: true
};
