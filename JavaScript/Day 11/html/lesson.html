Lesson

Day 11 — Events & Event Handling

addEventListener, the event object, preventDefault, mouse and keyboard events, bubbling, and delegation — with reference output per topic.

Topic 1 addEventListener

Pattern: element.addEventListener("eventName", handler). You can attach multiple listeners to the same event; use a named function reference if you plan to remove it later with removeEventListener.

                        Topic 2
                    

                    
                        The event object
                    

                

                


                    

                        Common properties
                    


                    
e.type              Event name ("click")
e.target            Element the event happened on
e.target.value      Current value (inputs)
e.target.textContent  Visible text (elements)
e.key               Key name (keyboard)
e.clientX / e.clientY  Pointer position in viewport


                    

                        Reference output (synthetic click)
                    


                    


                


            


            

            


                

                    
                        Topic 3
                    

                    
                        preventDefault & stopPropagation
                    

                

                


                    

                        Many elements have
                        default actions
                        (forms submit and navigate, links follow href, context
                        menu on right-click).
                        e.preventDefault()
                        cancels the default for that event.
                        e.stopPropagation()
                        stops the event from bubbling further (see Topic 6).
                    


                    

                        Reference output
                    


                    


                


            


            

            


                

                    
                        Topic 4
                    

                    
                        Mouse events
                    

                

                


                    

                        Cheat sheet
                    


                    
click        Single primary-button press + release
dblclick     Two rapid clicks
mouseover    Pointer enters element (or descendant)
mouseout     Pointer leaves element (or descendant)
contextmenu  Usually right-click — cancelable


                    

                        Reference output
                    


                    


                


            


            

            


                

                    
                        Topic 5
                    

                    
                        Keyboard & input
                    

                

                


                    

                        When to use which
                    


                    
input    Fires as the field changes — read e.target.value
keydown  Key pressed down — read e.key, modifiers; can prevent defaults
keyup    After key released


                    

                        Reference output
                    


                    


                


            


            

            


                

                    
                        Topic 6
                    

                    
                        Bubbling & event delegation
                    

                

                


                    

                        Most DOM events
                        bubble
                        : they fire on the target, then on each ancestor in order.
                        Delegation
                        means attaching
                        one
                        listener on a parent and branching on
                        e.target
                        (often with
                        closest("li")
                        in real apps). New children get the behaviour
                        for free.
