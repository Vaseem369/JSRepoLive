Day 4 — Control Flow

Topic 1 The if Statement — if (condition) { … }

Visual reference 

Syntax:   if (condition) { ...code to run if true }

Rule:   If the condition is true, the block runs. If false, skipped.

Always use curly braces — even for one-liners. Skipping them causes bugs
when you later add a second line.

Follow along — predict each output 

const age = 20;
if (age >= 18) console.log("vote");
→
your answer

const score = 30;
if (score > 50) console.log("pass");
→
your answer

                        Topic 2
                    

                    
                        if / else if / else — Decision chains
                    

                

                


                    

                        Order matters:
                        JavaScript stops at the
                        first
                        true branch. Order from most specific to least
                        specific.
                    


                    

                        Common bug:
                        writing
                        marks >= 60
                        before
                        marks >= 90
                        makes the 90+ branch unreachable (90 is also ≥ 60).
                    


                    

                        Visual reference
                    


                    
if (conditionA) {
  // branch A
} else if (conditionB) {
  // branch B
} else {
  // fallback
}


                    

                        Follow along — what grade does Aarav get?
                    


                    
marks = 85
if (marks >= 90) "A"
else if (marks >= 75) "B"
else if (marks >= 60) "C"
else "F"
→
grade?

marks = 50 → grade?
→
your answer

marks = 95 → grade?
→
your answer

                        Topic 3
                    

                    
                        Nesting & Why to Avoid It
                    

                

                


                    

                        Code that drifts right is a smell. The
                        pyramid of doom
                        —
                        if
                        inside
                        if
                        inside
                        if
                        — is allowed but hard to read. Each level adds
                        mental effort.
                    


                    

                        Example (hard to follow)
                    


                    
if (user) {
  if (user.isActive) {
    if (user.age >= 18) {
      console.log("Access granted");
    }
  }
}


                    

                        We flatten this with
                        guard clauses
                        in Topic 6.

                        Topic 4
                    

                    
                        switch Statement
                    

                

                


                    

                        Use when comparing
                        one
                        variable to many
                        exact
                        values (day name, status, role).
                    


                    

                        Visual reference
                    


                    
3 keywords:   case (match)   break (exit)   default (fallback)

⚠ Forget break → fall-through! Execution continues into the NEXT case.
The #1 switch bug.


                    

                        Follow along
                    


                    
day = "Tuesday"
switch (day) { case "Tuesday": "Second day"; ... }
→
output?

day = "Saturday"
case "Saturday":
case "Sunday": "Weekend"
→
output?

case "5":  ...  value = 5 (number)
does it match?
→
yes or no?

                        Topic 5
                    

                    
                        Truthy & Falsy (deep)
                    

                

                


                    

                        Critical — memorise this
                    


                    
The 6 falsy values:
  1. false    2. 0    3. "" (empty string)    4. null    5. undefined    6. NaN

Everything else is truthy.

Surprises:   "0" (string) → truthy   ·   [] → truthy   ·   {} → truthy   ·   " " (space) → truthy


                    

                        Follow along — truthy or falsy?
                    


                    
if (0) { ... }
→
runs?

if ("0") { ... }
→
runs?

if ([]) { ... }
→
runs?

if (null) { ... }
→
runs?

if (NaN) { ... }
→
runs?

if ("false") { ... }
→
runs?

                        Topic 6
                    

                    
                        Guard Clauses — Clean Code
                    

                

                


                    

                        Pattern:
                        handle invalid cases at the top with
                        return.
                        The happy path sits at the bottom with less nesting.
                    


                    

                        Before — nested
                    


                    
function checkAccess(user) {
  if (user) {
    if (user.isActive) {
      if (user.age >= 18) {
        console.log("Access granted");
      }
    }
  }
}


                    

                        After — guard clauses
                    


                    
function checkAccess(user) {
  if (!user) return;
  if (!user.isActive) return;
  if (user.age < 18) return;

  console.log("Access granted");
}
